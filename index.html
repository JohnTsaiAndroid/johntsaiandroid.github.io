  • View的测量流程，布局流程和绘制流程。

  • View的滑动

自定义View的几种类型:

  • 继承自View或ViewGroup
  • 继承自系统现有的控件

ViewRoot和DecorView

ViewRoot类对应于ViewRootImpl类，是连接WindowManager和DecorView的纽带，View的三大流程都是由ViewRoot来完成的。

View的绘制过程是从ViewRoot的performTraversals开始的，经过measure,layout和draw三个过程最终将一个View绘制出来。

  • measure测量View的宽和高
  • layout确定View在父容器中的放置位置
  • draw绘制View到屏幕上的

performTraversals方法会依次调用performMeasure,performLayout,performDraw三个方法（分别完成顶级View的测量，布局和绘制流程）。

测量流程:
performMeasure->measure->onMeasure(onMeasure方法会对多有子元素进行measure过程，此时measure流程从父容器中传递到子元素中)===>完成依次measure过程====>子元素重复递归父容器的measure过程，完成View树的遍历。

布局流程，绘制流程和测量流程类似，唯一的区别是，performDraw的传递过程是在draw方法中通过dispatchDraw来实现的。

使类和成员的可访问性最小化

模块设计好坏的最重要的因素是：该模块对于外部的其它模块而言，是否能隐藏它的内部数据和其它实现细节。

设计良好的模块:

  • 隐藏所有的实现细节(把它的API和它的实现隔离开)
  • 模块之间只通过API进行通信（模块不需要知道其他模块的内部工作情况）

上面那个概念叫做信息隐藏(information hiding),封装(encapsulation)

Java提供许多机制协助信息隐藏：

  • 访问控制机制(类，接口和成员的可访问性)
  • 实体的可访问性(实体声明所在的位置和声明中的访问修饰符)

尽可能使每个类或者成员不被外界访问

对于顶层的（非嵌套的）类和接口，两种可能的访问级别：

  • 包级私有的(package-private)
  • 公有的(public)

对于成员(域，方法，嵌套类和嵌套接口)有四种可能的访问级别:

  • 私有的(private)——只有在声明该成员的顶层类内部才可以访问这个成员
  • 包级私有的(package-private)——包内部的所有类
  • 受保护的(protected)——包内部的所有类和子类(有限制)
  • 公有的(public)——任何地方
  • 如果方法覆盖了超类中的一个方法，子类中的访问级别就不允许低于超类中的访问级别。
  • 不能为了测试，而将类，接口或者成员变成包的导出的API的一部分（可以让测试作为被测试的包的一部分来运行，从而能够访问它的包级私有元素)
  • 实例域决不能是公有的
  • 静态域也不能是公有的(例外情况：常量构成了类提供的整个抽象中的一部分，可通过公有的静态final域来暴露这些常量)

长度非零的数组总是可变的，客户端能够修改数组中的内容。（安全漏洞的一个常见来源）

在公有类中使用访问方法而非公有域

退化类的数据域可以直接访问，没有提供封装的功能。对于可变的类来说，应该用包含私有域和公有设值方法(setter)的类代替:

如果类可以在它所在的包得外部进行访问，就提供访问方法。

如果类是包级私有的，或是私有的嵌套类，直接暴露数据域并没有错误。

使可变性最小化

不可变类只是其实例不能被修改的类。常见的不可变类有:String,基本类型的包装类,BigInteger和BigDecimal等

每个实例中包含的所有信息都必须在创建该实例的时候提供，并在对象的整个生命周期内固定不变。

使类成为不可变的五条原则:

  • 1.不要提供任何会修改对象状态的方法(即改变对象属性的方法)
  • 2.保证类不会被拓展（final class）
  • 3.使所有的域都是final的
  • 4.使所有的域都是私有的
  • 5.确保对于任何可变组件的互斥访问

不可变对象本质上是线程安全的，不要求同步。不可变对象可以被自由地共享。不可变类应该重用现有的实例。对于频繁用到的值，为它们提供公有的静态final常量。

不可变对象为其他对象提供了大量的构件(building blocks)

不可变类真正唯一的缺点是对于每个不同的值都需要一个单独的对象。
执行一个多步骤的操作，每个步骤都会产生一个新的对象，除了最后的结果之外其他的对象最终都会被丢弃。导致性能问题暴露出来。

不可变类String类的可变配套类是StringBuilder

对于构建不可变的类，为了确保不可变性，类绝对不允许自身被子类化:

  • 1.使类成为final的
  • 2.让类的所有构造器都变成私有的或包级私有的，并添加公有的静态工厂方法代替公有的构造器

复合优于继承

继承：可实现代码重用，但并非永远是完成这项工作的最佳工具

安全的继承:

  • 包内继承
  • 专门为了继承而设计的，具有良好文档说明的类的继承

不安全的继承：

  • 对普通的具体类进行跨越包边界的继承

继承打破了封装性（即子类依赖于其超类中特定功能的实现细节）

解决方法：使用复合代替继承（不用拓展现有的类，而是在新的类中增加一个私有域，引用现有类的一个实例，现有类变成新类的一个组件）

只有当子类真正是超类的子类型(subtype)时，才适合用继承

对于类A和B，只有两者之间确实存在”is-a”关系的时候，类B才应该拓展类A。

继承机制会把超类API中的所有缺陷传播到子类中，而复合则允许设计新的API来隐藏这些缺陷。

结论：继承的功能非常强大，但存在很多问题。它违背了封装原则。只有当子类和超类之间确实存在子类型关系时，使用继承才是恰当的。若子类和超类处在不同的包中，并且超类并不是为了继承而设计的，那么继承将会导致脆弱性。可以用复合和转发机制代替继承，尤其是当存在适当的接口可以实现包装类的时候，包装类不仅比子类更加健壮，而且功能更加强大。

接口优于抽象类

Java中定义允许多个实现的类型：接口和抽象类

区别：

  • 抽象类允许包含某些方法的实现，但接口不允许。
  • 抽象类作为类型定义受到了极大的限制（为了实现由抽象类定义的类型，类必须为抽象类的一个子类，而接口不管这个类是否处于类层次的哪个位置）

实现新的接口，现有的类可以很容易被更新。

实现新的接口只需要在类的声明中增加一个implements子句，增加必要的方法。无法更新现有的类来拓展新的抽象类，两个类拓展同一个抽象类，必须把抽象类放到类型层次的高度，使这两个类的祖先先成为它的子类，会间接地伤害类层次。

接口是定义混合类型的理想选择。

类除了实现它的基本类型之外，还可以实现混合类型，表明它提供某些可选择的行为。

接口允许我们构造非层次结构的类型框架。

类型层次对于卒子某些事物非常合适，但是其他有些事物并不能被整齐地组织成一个严格的层次结构。

接口只用于定义类型

类实现接口，接口充当引用这个类的实例的类型。为了其他任何目的定义接口都是不恰当的。

例如常量接口(constant interface)

常量接口会暴露实现细节到该类的导出API中。它代表了一种承诺，即使以后不需要这些常量了，也必须实现这个接口（以确保二进制兼容性）。

导出常量的几种方式：

  • 将常量添加到类或接口中
  • 使用枚举类型
  • 使用不可实例化的工具类

类层次优于标签类

多种风格的实例的类，并包含实例风格的标签域。

优先考虑静态成员类

嵌套类(nested class):定义在另一个类内部的类，为它的外部类（enclosing class）提供服务。

若嵌套类将来可能用于其他的某个环境中，它就应该是顶层类，而不是嵌套类

四种嵌套类:

  • 1.静态成员类(static member class)
  • 2.非静态成员类(nonstatic member class)
  • 3.匿名类(anonymous class)
  • 4.局部类(local class)

静态成员类：是外部类的一个静态成员，是最简单的一种嵌套类。可将它看做是普通的类，只是声明在另一个类的内部而已。可以访问外部类的所有成员（包括私有成员）

非静态成员类和静态成员类的区别:

  • 静态成员类的声明中包含修饰符static
  • 非静态成员的每个实例都隐含着外部类的一个外围实例(enclosing instance)

匿名类:不同于Java中的其他任何语法单元。没有名字，不是外部类的一个成员。在使用的同时被声明和实例化。

匿名类的适用性受到很多限制:

  • 1.无法实例化
  • 2.不能执行instanceof测试或做任何需要命名类的其他事情
  • 3.无法实现多个接口或拓展一个类，或同时拓展类和实现接口。

匿名类的用法:

  • 动态创建函数对象
  • 创建过程对象(如Runnable,Thread,TimerTask实例)
  • 在静态工厂方法内部

局部类:用得最少。在任何可以声明局部变量的地方都可以声明局部类。
有名字（与成员类一样），必须非常简短，以便不影响可读性(与匿名类一样)，在非静态环境中定义的时候，才有外围实例，也不能包含静态成员（与匿名类一样）

View基础知识

主要内容有：

  • View的位置参数
  • MotionEvent和TouchSlop对象
  • VelocityTracker,GestureDetector和Scroller对象

什么是View

View是Android中所有控件的基类。View是界面层的控件的一种抽象。除了View之外，还有ViewGroup。它表示一组控件，也继承自View。

View的位置参数

如下图所示，View的位置主要由它的四个顶点来决定的，分别对应于View的四个属性：top,left,bottom,right.其中左上角坐标为(left,top),右下角为（right，bottom).

NOTE:这些坐标都是相对于View的父容器来说的，也就是说是一种相对坐标。

View的宽高和坐标之间的关系：

width = right - left;

height = bottom - top;

获取View的这四个属性的方式：

  • left = getLeft();
  • right = getRight();
  • top = getTop();
  • bottom = getBottom();

Android3.0开始，View新增了几个参数:x,y,translationX,translationY.

x,y是View左上角的坐标，translationX和translationY是View左上角相对于父容器的偏移量(默认值为0)。这几个参数都是相对于父容器的坐标。

这几个参数的转化关系:

x=left+translationX

y=top+translationY

View在平移过程中，top和left表示View原始左上角的位置信息，值不会改变，发生改变的是x,y,translationX和translationY这四个参数.

MotionEvent和TouchSlop

MotionEvent

在手指触摸屏幕后产生一系列事件中，典型的有以下几种

  • ACTION_DOWN(刚接触屏幕)
  • ACTION_MOVE(在屏幕上移动)
  • ACTION_UP（从屏幕上松开）

通过MotionEvent对象可以得到点击事件的坐标。
系统提供了两组方法：

getX/getY和getRawX/getRawY
getX/getY返回相对于当前View左上角的xy坐标
getRawX/getRawY返回的是相对于手机屏幕左上角的xy坐标

TouchSlop

TouchSlop是Android系统能识别出的滑动的最小距离。常量，具体值和设备有关。

获取方式：ViewConfiguration.get(getContext()).getScaledTouchSlop()

在Android源码中能找到TouchSlop这个常量的定义:https://github.com/android/platform_frameworks_base/blob/master/core%2Fres%2Fres%2Fvalues%2Fconfig.xml

VelocityTracker、GestureDetector和Scroller

VelocityTracker

速度追踪，用于追踪手指在滑动过程的速度.

Android API document:http://developer.android.com/reference/android/view/VelocityTracker.html

GestureDetector

手势检测,用于辅助检测用户的单击，滑动，长按，双击等行为.

Android API document:http://developer.android.com/reference/android/view/GestureDetector.html

Scroller

弹性滑动对象，用于实现View的弹性滑动

View实现滑动的两种方式及其对比：

  • View的ScrollTo/Scroll(滑动过程瞬间完成，无过渡效果，易造成滑动用户体验不好)
  • Scroller和View的computeScroll方法配合完成View的弹性滑动(非瞬间完成的，有过渡效果)

为Genymotion设置Charles代理

发表于 2015-12-15   |  

前言

Charles是Mac平台上一款非常优秀的HTTP网络抓包工具。

Android抓包设置Charles代理

为Genymotion设置代理

  • 1依次打开genymotion的设置–>WLAN
  • 2长按WirelessSSID，选择高级选项,将代理设置为手动
  • 3分别将代理服务器设为10.0.3.2（genymotion在本地网络环境下的IP），将端口号设置为8888
  • 4.保存即可

为Android真机设置代理

Mac查看本机IP地址：

  • 在终端中输入ifconfig
  • 系统偏好设置->高级->TCP/IP

具体步骤和Genymotion的类似，只是将代理服务器地址设为本机IP地址。

注意当真机请求时，Charles软件会弹出对话框，选择allow即可。
